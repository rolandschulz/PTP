Editing Connection Properties

Once a connection has been created it may still be necessary to define the port and other properties for the connection. This is a common cause of connection problems.

You can monitor and change the properties of your connection in the Properties view. To open this view from the workbench menu go to Window > Show View > Properties. You are able to change some values such as the description or the server name. Although each Remote System Explorer subsystem maintains its own list of properties, three properties (connected or disconnected, port, and user ID) are shared among all subsystems. If you change any of these properties in one subsystem, the other subsystems reflect the change.

Select a subsystem and check the Properties view to see the shared properties for all of your subsystems. For example, the Connected value is Yes or No for all of your subsystems under one connection.

It is also possible to view and edit the Properties of the subsystems using the Remote System Details view. To do this, right click on a connection in the Remote Systems view and select Show In Table.

In review, there are various methods for changing the properties for a connection:

  1. Right click on the connection name in the Remote Systems view and select Properties.
  2. Expand the connection so that the available subsystems are listed. To modify the properties specific to a particular subsystem right click on the subsystem and select Properties.
  3. Open the Properties view (Window > Show View > Properties). Select a connection or subsystem and its properties will be displayed in the Properties view.
  4. Right click on a connection in the Remote Systems view and select Show In Table. This will show all the properties of all the subsystems in the Remote System Details view. This view allows the port used by the various subsystems to be changed.
