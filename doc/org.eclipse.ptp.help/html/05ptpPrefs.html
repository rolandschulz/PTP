PTP Preferences

  • Runtime preferences
  • Debug preferences
  • Open RTE preferences
  • Viewer preferences

To open the Preferences window, select Window > Preferences... Then select 'PTP' in the list of preferences pages on the left side.

The main page has Runtime Preferences. Runtime systems will let you select between different types of external runtimes (e.g. Open MPI, MPICH, etc.) We have separated control and monitoring for situations where the runtime does not provide both services (e.g. MPICH). At the moment you can only choose 'Open Runtime Environment'.

There is a checkbox indicating if you want to enable Developer Mode. This allows use of the PTP Simulator. If you do this, you have to restart PTP to enable simulation mode. When you restart, you will be able to select the Simulation mode from the preferences page.

Debug Preferences - (PTP->Debug) Main Debug preferences page. These are preferences that apply to any debugger. 'Show full paths on breakpoint' will use the full path to the source file when displayed in the debugger Breakpoint View. 'Register process 0 by default' will register process 0 with the Debug View when the parallel debugger starts. Otherwise no processes will be registered.

PTP SDM (Scalable Debug Manager) Preferences - (PTP -> Debug -> SDM) These are Preferences specific to the SDM.

PTP->Open RTE: Preferences specific to the Open Runtime Environment. 'Open Runtime Environment PTP Proxy Server' specifies the location to use for the Open RTE proxy server, which is used for remote job launch and monitoring of system and job status. These preferences do not normally need to be changed.

PTP->Viewer: Preferences specific to the PTP viewing UI.
