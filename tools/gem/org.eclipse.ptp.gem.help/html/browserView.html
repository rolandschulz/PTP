Browser View

Activating the Browser View

When you formally verify an MPI C/C++ program, the Browser View and the Analyzer View will both be opened and depending on what is specified in preferences either one may be brought to the foreground and given the active focus.

Browser Overview

In the menu of this view there are three buttons

  • The Terminates the current operation
  • The Saves the results to a file
  • The Opens this help page

The purpose of this view is to summarize and categorically group all of the errors and warnings found. At the top left is a brief summary of the results indicating the number of errors and warnings found. In the top right are users are provided with the ability to set the number of processes and run ISP.

Below the summary is a section containing a tab for category of error or warning that was found in the code. By opening a given tab you are given detailed information on each occurrence of that type of error or warning in your code. Clicking on any of these occurrences will open the Eclipse editor with that line of code highlighted. If there are no instances of the problem then there is a short message indicating as much. Furthermore, a user is shown which types of errors are presented via the icons of the tab.

  • The indicates that there are no instances of that type of problem
  • The tells the user that this problem is present in their code.

Errors Tab

This tab is reserved for problems that cause the program to crash or deadlock.

Assertion Violations Tab

This tab list all local assertion violations found.

Leak Tab

This tab list all MPI resource leaks found.

Irrelevant Barriers Tab

This tab is reserved for Irrelevant Barriers or barriers that can be safely removed without changing the runtime behavior of the program (aside from possibly speeding the program up a little).
