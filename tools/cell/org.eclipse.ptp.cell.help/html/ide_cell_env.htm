Cell Environments

There are 4 types of cell environments that can be used to run and debug cell applications:
  1. Attached Cell Simulator - Cell simulator running on a remote machine. The simulator must be manually started on the remote machine, as opposed to the Remote Cell Simulator type, where the Cell IDE handles the starting of the simulator on the remote host.
  2. Cell Box - Physical Cell machine (remote or native cell machine running the Cell IDE).
  3. Local Cell Simulator - Cell simulator running on the native machine.
  4. Remote Cell Simulator - Cell simulator running on a remote machine. The Cell IDE handles starting the remote simulator, as opposed to the Attached Cell Simulator type, where the simulator must be manually started on the remote host.
Prior to running/debugging a cell application, a cell environment must be created and started:
  • Open the Cell Environments view.
  • Right click on any of the 4 available cell machine types and select Create.
  • Fill in any necessary configuration values and click Finish.
  • Start the environment: Click on the green Start arrow button.
  • After the cell environment has started, it is ready to be used for cell application deployment.
