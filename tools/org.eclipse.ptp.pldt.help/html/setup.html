Setup for PTP MPI Tools

Launch Eclipse with the PTP Parallel Language Development Tools feature.

The CDT (or FDT) plug-ins are also required.

  1. Create a CDT project (e.g. Managed Make C Project)
  2. Create a source file e.g. test.c with MPI artifacts (see a small sample MPI program here: testMPI.c).
  3. Set up the include paths etc:
    Project Properties, C/C++ Build, Configuration Settings, Tool Settings tab, Directories
    --> Add include path for mpi header files etc. - path to the 'include' dir

    Project Properties, C/C++ Build, Configuration Settings, Tool Settings tab, GCC C Linker, Libraries

    1. Under "Libraries (-l)" add the symbol "mpi"
    2. Under "Library search path (-L)" add the path to the 'lib' dir in MPI dir.

  4. Make sure the Indexer is turned on
    Project Properties, C/C++ Indexer, select "Full C/C++ Indexer (declarations and cross references)"
    And also the same in Window, Prefs, C/C++, Indexer (this seems to be the default setting for new projects)

  5. Tell the plug-in where to find the MPI header file

    Window, Preferences, MPI
    Add path to MPI include paths (again) - this tells it what symbols to look for in the MPI Artifact view.

Now set up to run the MPI plug-in
